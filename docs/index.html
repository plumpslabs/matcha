What is matcha?

Every AI coding session starts the same: jump in, write code, realize mid-way you missed something obvious. matcha is a 6-checkpoint filter that runs before, during, and after every implementation — catching the things you'd catch yourself if you stopped to think. No bloat. Just a deliberate gate between you and messy code.

DNA — The Four Tenets

🧹 Simple

Minimum moving parts. One function = one thing. No unnecessary abstractions. If it feels complex, it's wrong.

⚡ Efficient

No filler. No dead code. Every line earns its place. Choose the shortest correct path — every time.

🎯 Deliberate

Think before you code. 5W1H check before starting. Explicit errors. Named constants. No guesswork.

♻️ Never Twice

Search before you write. Reuse, don't rewrite. Single source of truth — in code, config, and docs.

🛡️ Kuma — MCP Safety Toolkit

Kuma is the safety layer that pairs with matcha. While matcha optimizes how you code, Kuma protects what you execute — blocking dangerous commands before they reach the OS. Together, they form a complete guardrail: matcha on the front end (conventions), Kuma on the back end (execution).
